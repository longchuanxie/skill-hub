# SkillHub 未来发展规划

## 一、愿景定位

**SkillHub → AI 技能的包管理平台**

让开发者可以像使用 npm 一样，发现、发布、管理 AI 技能包（Skills/Prompts）。

---

## 二、npm 发布信息

| 项目 | 内容 |
|------|------|
| 包名 | `@xielc/skillhub` |
| 作用域 | `@xielc` |
| 安装命令 | `npm install -g @xielc/skillhub` |
| 源码 | [github.com/xielc/skill-hub](https://github.com/xielc/skill-hub) |

---

## 三、发展阶段

### 第一阶段：基础平台化

**目标**：建立技能市场的基础能力

| 功能模块 | 核心功能 | 说明 |
|----------|----------|------|
| **多格式兼容** | 自动识别技能包格式 | 支持 ZIP、Claude Skills 等多种格式，无需强制 skill.json |
| **市场基础** | 公共发布与发现 | 发布技能到公共市场，支持搜索/分类/标签过滤 |
| **版本管理** | 已有能力复用 | 延续现有的版本控制、在线预览测试能力 |
| **统计与评价** | 下载量、评分、评论 | 建立技能的质量信号 |

**技术重点**：
- 技能包自动解析（提取元信息、分类）
- 市场 API 设计

---

### 第二阶段：生态扩展

**目标**：构建平台的网络效应

| 功能模块 | 核心功能 | 说明 |
|----------|----------|------|
| **依赖管理** | 技能间依赖声明 | 支持技能声明对其他技能的依赖 |
| **CLI 工具** | `skill-cli` | 本地安装、发布、管理技能 |
| **企业私有仓库** | 已有能力增强 | 强化权限管理、审计日志 |
| **创作者经济** | 技能评分排行 | 建立创作者榜单 |

**技术重点**：
- 依赖解析与安装逻辑
- CLI 工具开发

---

### 第三阶段：商业化与扩展

**目标**：实现商业闭环

| 功能模块 | 核心功能 | 说明 |
|----------|----------|------|
| **MCP 支持** | MCP 服务器市场 | 扩展支持 MCP 类型 |
| **托管服务** | SkillHub Cloud | 托管+订阅模式 |
| **插件市场** | 第三方扩展 | 支持审核插件等扩展 |

---

## 三、关键技术决策

| 决策点 | 选择 | 理由 |
|--------|------|------|
| 技能格式 | 多格式兼容 | 不强制标准，降低用户迁移成本 |
| 协议支持 | Skill 优先 | 第一阶段聚焦，MCP 作为后续扩展 |
| 分发方式 | npm 风格 CLI + Web | 兼顾本地和浏览器场景 |
| 依赖管理 | 可选依赖声明 | 非强制，保持灵活性 |

---

## 四、成功指标

| 阶段 | 指标 |
|------|------|
| 第一阶段 | 公共市场上线，月活 100+ 技能发布 |
| 第二阶段 | CLI 安装量 1000+，企业客户 10+ |
| 第三阶段 | MCP 支持上线，付费客户 50+ |
